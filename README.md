# alexandria-spanish-slm
This repo contains everything related to the development of Alexandria. A small language model made for learning purposes on transformers and language modeling.

---

Alexandria is a decoder-only transformer model for language modeling in Spanish, trained on the custom dataset [tiny-coop-es](https://huggingface.co/datasets/hetline/tiny-coop-es). 

My main goal on this project was to learn deeply about language modeling, specially focused on the pre-training phase with a small language model. That's why I decided to implement almost all the main components by hand, as the main goal was understanding all the real challenges behind this kind of tasks.

Creating Alexandria gave me the opportunity to implement the next components, and I'm open sourcing everything:

1. A custom BPE tokenizer
2. A custom synthetic dataset with fables in Spanish: [tiny-coop-es](https://huggingface.co/datasets/hetline/tiny-coop-es)
3. A custom decoder-only transformer model named Alexandria at version 1.1

I'm deeply proud of this project given all I could not only learn, but also document on my thought process and my desicions on the journey.

## Quickstart

Alexandria will be shared via HuggingFace with Transformers, but this is still pending work. For now you can access it throught the code in this repository. Weights are open-sourced as well.

### Installation

This project manages the environment using uv, so make sure you have installed it.

```bash
uv sync
uv pip install -e .
```

### The config file

All the relevant parameters used in AlexandriaTokenizer, AlexandriaModel and the training pipeline are directly defined in the **config.yaml** file.

### Training the tokenizer

By default AlexandriaTokenizer trains using a tiny-stories-es, a custom synthetic dataset that will be released soon. Note this is different to [tiny-coop-es](https://huggingface.co/datasets/hetline/tiny-coop-es), which is the data used to train Alexandria. 

If you want to know more about why this works like this, please refer to [Data and Tokenizer design](https://github.com/carlos-ochoa/alexandria-spanish-slm/blob/main/docs/data_design_choices.md).

```bash
uv run src/tools/data/train_tokenizer.py
```

### Tokenizing the training data for the model

By default the script executes tokenization over [tiny-coop-es](https://huggingface.co/datasets/hetline/tiny-coop-es). 

```bash
uv run src/tools/data/dataset_tokenization.py
```

### Train Alexandria language model

By default it trains using **tokenized_docs.pt** file, generated by the previous step.

```bash
uv run src/training_pipeline.py
```

### Running inference

You can use any model version (1 or 1.1) to compare with temperature sampling, varying the values. Currently, no top-p or top-k methods are implemented. 

If you're curious on the differences between version 1 and 1.1 please refer to [Alexandria Model Card](https://github.com/carlos-ochoa/alexandria-spanish-slm/blob/main/docs/model_design_choices.md).

```bash
uv run src/run_inference.py
```
